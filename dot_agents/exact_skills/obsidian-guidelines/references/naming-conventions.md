# Naming Conventions

## Dates

Always use `YYYY-MM-DD` format everywhere:
- Daily notes: `Daily/YYYY-MM-DD.md`
- Dated journal entries: `YYYY-MM-DD - Event.md` (in root)
- Dated references: `YYYY-MM-DD - Article Title.md` (in Clippings/)

## Categories and Tags

Always pluralize:
- ✅ `Books`, `Movies`, `People`, `Places`, `Podcasts`
- ❌ `Book`, `Movie`, `Person`, `Place`, `Podcast`

This collapses hundreds of future decisions into one rule.

## Reference Note Filenames

Name reference notes using the actual title:
- ✅ `Marcus Aurelius.md`
- ✅ `Meditations.md`
- ❌ `marcus-aurelius.md`
- ❌ `meditations-book.md`

## Markdown Standards

- Use standard Markdown only
- Prefer wiki links `[[Note Name]]` for internal linking
- Use standard frontmatter YAML
- Avoid syntax that won't render outside Obsidian

## Ratings

When rating books, movies, or other media, use the 7-point scale:

| Rating | Meaning |
|--------|---------|
| 1 | Terrible |
| 2 | Very bad |
| 3 | Bad |
| 4 | Mediocre |
| 5 | Good |
| 6 | Very good |
| 7 | Excellent |

This avoids the clustering problem of 5-star scales.
