# Linking Practices

## Core Rule

Use internal links profusely. Try to always link the first mention of something in a note.

## Why

Links create the emergent structure of a bottom-up vault. Without links, notes become
isolated silos. With links, connections form naturally and new insights emerge from
unexpected juxtapositions.

## Practices

### First Mention Linking

In any note, the first mention of a person, place, book, concept, or idea should be a wiki link:

```markdown
I was reading [[Meditations]] by [[Marcus Aurelius]] and it reminded me of [[Stoicism]].
```

Subsequent mentions in the same note don't need to be linked unless it adds clarity.

### Journal as Link Hub

Journal entries are often the most heavily linked notes. A typical journal entry links to:
- People met or thought about
- Books being read
- Places visited
- Concepts or ideas encountered
- Projects or tasks worked on

This turns a journal from a chronological log into a network entry point.

### Backlinks as Navigation

Use the backlink panel to discover connections you didn't explicitly make. When you
mention something without linking, Obsidian shows it as an unlinked mention — consider
turning these into explicit links.

### Link Before You Think

When writing, link first, organize later. Don't stop to decide if a link is "important enough."
The cost of an extra link is near zero; the cost of a missing link is lost connections.

### Avoid Over-Linking

Don't link generic words or common terms that don't refer to specific notes.
Link proper nouns, specific concepts, and note-worthy ideas.

## External Links

Prefer internal wiki links `[[Note]]` over external URLs when a note exists in the vault.
Use external links only for sources, references, and things that don't warrant a vault note.
